Binary Search in Java: A Complete Tutorial

Learn how to implement Binary Search with both Iterative and Recursive approaches

Introduction to Binary Search

Binary Search is a fast and efficient algorithm for finding an item from a sorted list. It works by repeatedly dividing the search space in half until the target element is found or the search space is exhausted. The time complexity of Binary Search is O(log n), making it much faster than linear search for large datasets.

Algorithm Explanation

The binary search algorithm follows these basic steps:

  • Start with two pointers: left and right, initially pointing to the first and last index of the array.
  • Calculate the middle index: middle = left + (right - left) / 2.
  • If the middle element matches the target, return the middle index.
  • If the middle element is larger than the target, move the right pointer to middle - 1.
  • If the middle element is smaller than the target, move the left pointer to middle + 1.
  • Repeat until the target is found or the pointers cross.

Iterative Binary Search in Java

Here’s an implementation of Binary Search using an iterative approach:

public class BinarySearch {

    // Iterative Binary Search Method
    public static int binarySearch(int[] arr, int target) {
        int left = 0;
        int right = arr.length - 1;

        while (left <= right) {
            int middle = left + (right - left) / 2;

            // Check if the target is at the middle
            if (arr[middle] == target) {
                return middle;  // Element found
            }

            // If the target is smaller than the middle element
            if (arr[middle] > target) {
                right = middle - 1;  // Search the left half
            } else {
                left = middle + 1;  // Search the right half
            }
        }

        return -1;  // Element not found
    }

    public static void main(String[] args) {
        int[] arr = {1, 3, 5, 7, 9, 11, 13, 15, 17};
        int target = 7;

        int result = binarySearch(arr, target);

        if (result == -1) {
            System.out.println("Element not found");
        } else {
            System.out.println("Element found at index " + result);
        }
    }
}

The binarySearch method works by narrowing down the search space from both ends and calculating the middle index in each iteration until it finds the target or concludes the target is absent.

Recursive Binary Search in Java

Here’s how you can implement Binary Search using recursion:

public class BinarySearch {

    // Recursive Binary Search Method
    public static int binarySearch(int[] arr, int left, int right, int target) {
        if (left > right) {
            return -1;  // Base case: Element not found
        }

        int middle = left + (right - left) / 2;

        // Check if the target is at the middle
        if (arr[middle] == target) {
            return middle;  // Element found
        }

        // If the target is smaller than the middle element
        if (arr[middle] > target) {
            return binarySearch(arr, left, middle - 1, target);  // Search the left half
        } else {
            return binarySearch(arr, middle + 1, right, target);  // Search the right half
        }
    }

    public static void main(String[] args) {
        int[] arr = {1, 3, 5, 7, 9, 11, 13, 15, 17};
        int target = 7;

        int result = binarySearch(arr, 0, arr.length - 1, target);

        if (result == -1) {
            System.out.println("Element not found");
        } else {
            System.out.println("Element found at index " + result);
        }
    }
}

The recursive version of Binary Search functions similarly to the iterative version. However, the search space is reduced through recursive calls rather than a loop. The binarySearch method is called again on either the left or right half of the array, depending on the comparison of the middle element with the target.

Time Complexity of Binary Search

The time complexity of Binary Search is O(log n), where n is the number of elements in the array. This is because, in each step, the size of the search space is halved, making Binary Search much faster than linear search, which has a time complexity of O(n).
